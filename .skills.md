# Skills: Control de versiones automático

## Descripción
Esta skill establece que por cada commit y push realizado en el proyecto, se debe registrar un resumen del cambio en el archivo `CONTROL_VERSIONES.md`.

## Reglas
- Cada vez que se realice un commit y push, se debe agregar una entrada en `CONTROL_VERSIONES.md` con:
  - Número de versión (ejemplo: V3.1)
  - Mensaje del commit
  - Breve descripción del cambio
  - Fecha y hora (opcional)
- El archivo `CONTROL_VERSIONES.md` debe mantenerse actualizado y versionado junto con el código fuente.
- Si el commit corresponde a un hotfix, mejora visual, refactorización o despliegue, debe especificarse en la descripción.

## Ejemplo de entrada
```
## V3.1
- CSS limpio y listo para build en Vercel
- Se eliminaron conflictos y se validó la compatibilidad para despliegue en producción.
- Commit y push realizados el 2026-04-12.
```

## Objetivo
Mantener un historial claro y auditable de los cambios importantes del proyecto, facilitando el seguimiento y la trazabilidad para cualquier miembro del equipo.
